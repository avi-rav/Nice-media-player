What it does

Everything you'd expect — done right.

From a debounced search pipeline to a fully keyboard-driven player and persistent watch history, every feature is built with performance and resilience in mind.

🎬

Custom Video Player

Hand-built controls with play/pause, seek bar, volume, fullscreen and on-the-fly quality switching that preserves your playback position.

⌨️

Keyboard First

Space/k to toggle, arrows to seek and adjust volume, m to mute, f for fullscreen — all with proper ARIA labels.

🔍

Debounced Search

300ms debounce → distinctUntilChanged → switchMap cancels in-flight requests. Client-side matching on title & author with no extra calls.

🎚️

Resolution Filtering

Buckets videos by true max resolution derived from pixel height — not the unreliable API quality field — updating instantly via computed signals.

🕒

Watch History

Persisted to localStorage with resume position and finished/stopped status. Play again, resume, or clear — sorted most-recent first.

🛡️

Defensive Mapping

Untrusted API data is normalized through a mapper that skips broken records instead of crashing the page. Graceful degradation throughout.

🔁

Smart Retries

HTTP interceptor retries only transient 429/5xx failures with exponential backoff, and fails fast on 4xx where retrying is pointless.

🔔

Toast Notifications

Signals-based notification queue rendered in an aria-live region, with friendly messages mapped from raw HTTP errors.

🔐

Secure by Design

The Pexels API key is injected server-side by a proxy/BFF — never bundled into the client. Runtime config token seeds tokens without code changes.

Single responsibilities

Services

Each service does exactly one thing well — from a thin HTTP client to pure resolution math, with orchestration kept separate from I/O.

PexelsApiService

Thin HTTP layer. Calls popular(), search() and getById() and returns raw DTOs only — no mapping, no error handling.

VideoCatalogService

Orchestration. Chooses popular-vs-search based on the query, applies the mapper, and returns normalized Video[] domain objects.

VideoMapperService

Defensive DTO→domain normalization. Handles null/malformed fields, derives resolution labels, filters to playable mp4 sources and dedupes by label.

ResolutionService

Pure resolution logic. Maps pixel height to labels (4K → 360p), computes distinct available resolutions and filters videos by bucket.

NotificationService

Signals-based notification queue. notify(), error(), dismiss() with auto-dismiss and a read-only signal view.

ErrorMapperService

HTTP error translation. Turns status codes (401, 429, 5xx…) into clear, user-friendly messages for the toast system.

Typed end-to-end

Strict separation of DTOs & domain models.

The wire format never leaks into the UI. Raw Pexels DTOs are mapped into clean domain models the rest of the app can trust.

📡

API DTOs (wire format)

PexelsVideoDto, PexelsVideoFileDto, PexelsVideoPictureDto, PexelsUserDto, PexelsVideoListResponseDto — nullable, unreliable fields treated as untrusted input.

✅

Domain Models (app-facing)

Video, VideoSource, ResolutionLabel, HistoryEntry, HistoryStatus — normalized, non-null where it matters, with a playable flag.
